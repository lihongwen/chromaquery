# 滚动行为测试指南

## 🎯 测试目标
验证新查询消息是否正确显示在对话框可视区域的顶部，而不是滚动到对话历史的开头。

## 📋 测试步骤

### 1. 准备测试环境
1. 启动ChromaDB Web Manager前端和后端
2. 进入查询页面
3. 选择一个有数据的集合

### 2. 创建长对话历史
1. 发送多个查询（至少5-10个），创建足够长的对话历史
2. 确保对话内容超过一屏，需要滚动才能看到所有消息
3. 手动滚动到对话历史的中间位置

### 3. 测试新查询滚动行为
1. 在输入框输入新的查询内容（例如："测试滚动行为"）
2. 按回车发送查询
3. **观察滚动行为**：
   - ✅ 正确：页面自动滚动，新发送的"测试滚动行为"消息显示在对话框最上方
   - ❌ 错误：页面滚动到对话历史的最开头，看不到刚发送的消息

### 4. 验证查询结果显示
1. 等待查询完成
2. 确认查询结果也显示在可视区域内
3. 用户应该能立即看到自己的查询和系统回复

### 5. 测试对话切换行为（保持不变）
1. 切换到其他对话
2. 确认滚动到该对话的开头（第一条消息）
3. 这个行为应该保持不变

## 🔍 调试信息
如果滚动行为不正确，可以：
1. 打开浏览器开发者工具的控制台
2. 查看滚动相关的调试日志：
   - "滚动到最新消息: [element]"
   - "latestMessageRef.current 为空"
3. 检查是否有JavaScript错误

## 🎨 期望的用户体验

### 正确的行为流程：
1. 用户输入查询 → 按回车
2. 新消息添加到对话底部
3. **页面自动滚动，让新消息显示在对话框顶部**
4. 用户立即看到自己的查询内容
5. 查询结果返回后，也在可视区域内显示

### 错误的行为（需要修复）：
1. 用户输入查询 → 按回车
2. 新消息添加到对话底部
3. **页面滚动到对话历史的最开头**
4. 用户看不到刚发送的消息，需要手动滚动到底部

## 🛠️ 技术实现验证
- `latestMessageRef` 应该正确指向最新消息的DOM元素
- `scrollToLatestMessage()` 函数应该计算正确的滚动位置
- 滚动应该是平滑的，使用 `behavior: 'smooth'`
- 延迟时间应该足够让DOM完全渲染（当前设置为300ms）
